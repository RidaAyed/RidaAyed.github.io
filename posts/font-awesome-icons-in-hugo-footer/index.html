Font awesome icons in hugo footer

5 May, 2024
Table of Contents

I followed this concise blog post1 about embedding font awesome in hugo. Of course I expected everything to work flawless in my environment as well. As usual, this presumption was wrong.

Simply adding {{< fa bath >}} to the footer key in config.toml nor in this org mode file will not work.

This post shall show how to circumvent that flaws, use fontawesome in hugo’s footer section and overall in your blog’s content using ox-hugo.

Directory path and completeness

Opposed to the instructions2 I put the files not in the static but in the assets folder. I had to create it first.

Furthermore I put the complete Fontawesome directory, not only a particular file. fa.html3 pointing to subdirectories made me skeptic about picking only particular files.

config.toml

The js key is documented in my theme’s repository4. You might encounter other ways for your particular theme in order to customize css/js assets.

[params]
js = ["/fontawesome/js/all.js"]
footer = '''
<a href="https://instagram.com/rida.ayed"><div class="fab fa-fab fa-instagram"></div></a>
<a href="https://github.com/RidaAyed"><div class="fab fa-fab fa-github"></div></a>
'''

Usage of the icons as ox-hugo shortcodes

Although being documented5, I rather show the shortcode usage in ox-hugo again and specifically:

@@hugo:{ {< fa bath >}}@@

Only by chance I noticed that surrounding the shortcode with the equal sign “=” works as well. Like

={ {< fa bath >}}=

You must delete that first space between the curly brackets from my upper examples. So {{ instead of { {

Conclusion

I showed how to extend the icon usage into the footer section of a hugo website. Furthermore you should be able to use the icon shortcodes in a org file considering ox-hugo as blog platform.
