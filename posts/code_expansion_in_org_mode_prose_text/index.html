Code expansion in org mode prose text

20 October, 2020

Say I want to write about the count of my system’s processors at the time of my writing. Or my current location’s weather or stock shares or …use your imagination. Now I do it like that:

At this very moment I’m writing this text on a machine with 4 processors.

In verbatim it’s written like this:

At this very moment I'm writing this text on a machine with src_bash{cat /proc/cpuinfo | grep processor | wc -l} {{{results(=4=)}}} processors.

Practically I expand a snippet, write any shelly command into the curly braces and instantly have it’s evaluated result written into my prose.

Way more elegant than my old way doing it like this:

At this very moment I’m writing this text on a machine with this count of processors:

cat /proc/cpuinfo | grep processor | wc -l

4

In verbatim it’s written like this:

At this very moment I'm writing this text on a machine with this [[src_0_b4a37322-bd9c-498c-91ef-233c5680433a][count]] of processors:

#+NAME: src_0_b4a37322-bd9c-498c-91ef-233c5680433a
#+begin_src sh :exports output
cat /proc/cpuinfo | grep processor | wc -l
#+end_src

#+RESULTS:
: 4
